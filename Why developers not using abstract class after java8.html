Abstract Classes vs Interfaces in Java

Why Developers Prefer Interfaces Over Abstract Classes

1️⃣ Java Now Supports Default Methods in Interfaces (Since Java 8)

Before Java 8, interfaces could only have abstract methods. But Java 8 introduced default methods.

interface Vehicle { void start(); default void stop() { System.out.println("Vehicle is stopping..."); } } class Car implements Vehicle { @Override public void start() { System.out.println("Car is starting..."); } }

2️⃣ Multiple Inheritance Support in Interfaces

interface Engine { void start(); } interface Transmission { void shiftGear(); } class Car implements Engine, Transmission { @Override public void start() { System.out.println("Car engine starting..."); } @Override public void shiftGear() { System.out.println("Car shifting gears..."); } }

3️⃣ Better Scalability and Loose Coupling

interface PaymentGateway { void processPayment(double amount); } class PayPal implements PaymentGateway { @Override public void processPayment(double amount) { System.out.println("Processing $" + amount + " via PayPal."); } }

4️⃣ Abstract Classes Are Less Flexible

abstract class Animal { abstract void makeSound(); } class Dog extends Animal { @Override void makeSound() { System.out.println("Dog barks!"); } }

5️⃣ Functional Interfaces & Lambda Expressions (Since Java 8)

interface Greeting { void sayHello(); } public class Main { public static void main(String[] args) { Greeting greet = () -> System.out.println("Hello, World!"); greet.sayHello(); } }

6️⃣ Interfaces Work Well with Dependency Injection

interface Database { void connect(); } class MySQLDatabase implements Database { @Override public void connect() { System.out.println("Connecting to MySQL..."); } }

7️⃣ Abstract Classes Still Have Their Uses

abstract class Vehicle { int speed; void showSpeed() { System.out.println("Speed: " + speed + " km/h"); } abstract void start(); }

Final Comparison: Abstract Class vs Interface

Feature Abstract Class Interface
Methods Can have both abstract and concrete methods Can have abstract, default, and static methods
Instance Variables Allowed Not allowed (only constants)
Multiple Inheritance Not supported Supported

Conclusion

In modern Java, interfaces have largely replaced abstract classes because they provide:

  • Greater flexibility
  • Multiple inheritance
  • Compatibility with newer Java features

However, abstract classes are still useful when you need shared state (fields) and partially implemented behavior.
