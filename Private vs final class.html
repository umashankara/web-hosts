Difference Between Private Class and Final Class in Java

1. Private Class

A private class means the class is only accessible within the enclosing class.

  • A top-level class cannot be private in Java.
  • Only inner classes can be private.

Example of a Private Inner Class:

class Outer {
    private class Inner {
        void display() {
            System.out.println("Inside private Inner class");
        }
    }
    
    void createInner() {
        Inner obj = new Inner();
        obj.display();
    }
}

2. Final Class

A final class cannot be inherited (extended) by any other class.

  • Used to prevent subclassing.
  • Ensures class behavior remains unchanged.

Example of a Final Class:

final class FinalClass {
    void show() {
        System.out.println("Inside final class");
    }
}

Key Differences

Feature Private Class Final Class
Definition A class that is accessible only within its enclosing class A class that cannot be extended (inherited)
Scope Only applies to inner (nested) classes Applies to both inner and outer classes
Inheritance Can be inherited if accessible Cannot be inherited
Use Case Restrict access within a specific class Prevent modification by inheritance
Allowed at Top-Level? ❌ No ✅ Yes

When to Use Which?

  • Use private for inner classes when you want to encapsulate them and restrict access.
  • Use final for classes that should not be extended, like utility classes (java.lang.String, java.lang.Math).
